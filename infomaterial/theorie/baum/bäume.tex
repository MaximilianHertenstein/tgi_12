\section{Knoten}
Ein Binärbaum besteht aus Knoten, die einen Inhalt sowie zwei, einen oder keinen Nachfolger haben.
\begin{minted}{java}
record Node<T>(T content, Node<T> left, Node<T> right) {}
var three = new Node<>(3, null, null);
var six = new Node<>(6, null, null);
var myNode = new Node<>(5, three, six);
\end{minted}

Beim Definieren einer Methode, die aus allen Werten im Binärbaum ein Ergebnis berechnet,
muss man diese Methode rekursiv auf den Nachfolgern eines Knotens aufrufen.
\begin{minted}{java}
class Tree {
    int sum(Node<Integer> node) {
        if (node == null) {
            return 0;
        } 
        else {
            int sumLeft = sum(node.left());
            int sumRight = sum(node.right());
            return node.content() + sumLeft + sumRight;
        }
    }
}
\end{minted}

\section{Wichtige Begriffe}
\begin{itemize}
  \item Den Anfangsknoten nennt man \textbf{Wurzel} (Ebene~0).
  \item Die direkt verbundenen Nachfolger eines Knotens nennt man \textbf{Kinder} oder \textbf{Kindknoten}.
  \item Den direkten Vorgänger eines Knotens nennt man \textbf{Elternknoten}.
  \item Ein Knoten ohne Kinder heißt \textbf{Blatt} oder \textbf{äußerer Knoten}.
  \item Ein Knoten mit mindestens einem Kind heißt \textbf{innerer Knoten}.
  \item Ein \textbf{Teilbaum} ist ein Baum, der bei einem Knoten beginnt, der nicht die Wurzel ist.
  \item Die \textbf{Ebene} eines Knotens gibt an, wie weit er von der Wurzel entfernt ist. Die Wurzel hat Ebene~0.
\end{itemize}

\section{Bäume}

Ein Baum ist entweder leer oder hat einen Anfangsknoten.

class IntTree {
